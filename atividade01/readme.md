# Atividade 01: Definição de Processo de Software

## 1. Contexto e Problema

Este projeto simula a operação de uma Startup, contratada para desenvolver uma aplicação mobile de **Gerenciamento de Sistema Acadêmico**. O desafio consiste em equilibrar a agilidade necessária para uma startup com a robustez exigida por sistemas educacionais.

## 2. Metodologia

Foi adotado um **Modelo de Processo Híbrido**, utilizando conceitos de metodologias ágeis (Sprints e MVPs) integrados a uma estrutura formal de definição de atividades para garantir a **redução de danos** e previsibilidade de entrega.

## 3. Definição do Processo

O processo está estruturado nos seguintes pilares:

- **Atividades:** Divisão lógica das etapas de produção.
- **Artefatos (Entrada/Saída):** Documentação e produtos gerados em cada etapa (Ex: Backlog, Protótipo, Build).
- **Recursos:** Definição de capital humano (PO, Dev, Designer) e ferramentas tecnológicas.
- **Procedimentos:** O "como fazer" técnico para garantir a padronização.

## 4. Estrutura do Repositório nesta pasta

- `processo.md`: Contém a definição textual completa e o código Mermaid.js para o diagrama.
- `entrega_final.pdf`: Arquivo exportado para submissão no SIGAA, contendo a visão consolidada do processo.

## 5. Papéis na Startup

- **Product Owner (PO):** Responsável pela interface com o cliente e priorização do Backlog.
- **Desenvolvedor Fullstack:** Responsável pela implementação técnica e integração (Docker/FastAPI/PostgreSQL).
- **UX/UI Designer:** Focado na prototipagem e validação da experiência do usuário.
